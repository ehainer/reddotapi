#initialize(params, options = {}, *args, &block) ⇒ Object

Build the search object that defines what to be

searched and any filters to be applied

Examples:

Basic search

# Passing the whole +params+ object, it will be sanitized automatically
# params = ActionController::Parameters.new({ search: { q: '*', page: 1 } })
@search = SearchService.new(params)
@search.results.each { |result| ... }

# Providing a Hash of parameters
@search = SearchService.new({ q: '*', page: 2, per_page: 25 })
@search.results.each { |result| ... }

Searching on specific models (User and Facility)

@search = SearchService.new({ q: '*' }, search: [User, Facility])

# Results will only contain matching User/Facility records
@search.results.each { |result| ... }

Searching with additional filters applied (data from arguments)

@search = SearchService.new({ q: '*' }, search: User, @user, facility) do |user, facility_id|
  with(:user_id) { user.id }
  boost(2.0) { with(:facility_id, facility_id) }
end

Parameters:

  • params (ActionController::Parameters, Hash)
  • options (Hash) (defaults to: {}) —

    Optional hash of search specific keys

  • args (Array) —

    Any number of arguments to be passed to block

  • block (Proc) —

    An optional block that will be called when searching

#documents ⇒ Object

Perform the search and assign the returned results

to the +@documents+ instance var. Normally one
should not call this method directly, but it will
return the raw, undecorated results from the search
if that is what is needed.

Examples:

Return the total number of search results (ignoring pagination)

@search = SearchService.new({ q: '*' })
total_results = @search.documents.hits.length

def documents
  @documents ||= Sunspot.search(searchable) do
    fulltext query
    paginate search_pagination
    order_by *search_sort

    # Call the block if given, which allows for further filtering/overriding
    instance_exec(*args, &block) if block.present?
  end
end
